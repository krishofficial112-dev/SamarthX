# Design Decisions

## 1. Structured dashboards

High-density operational data is divided into summary metrics, analytical cards, filters, quick insights, recent activity, and tabular records. This reduces scanning effort and keeps common actions visible.

## 2. Scope-based permission mapping

The permission interface follows a progressive hierarchy:

**Districts → Blocks → Clusters → Schools → Permissions**

Selection summaries and chips keep the current scope visible. Search, filters, pagination, and bulk selection make the layout suitable for large datasets.

## 3. First-time applicant access

The admission portal gives new users direct access to registration, application guidance, instructions, and support before login. This reduces dependency on hidden post-login information.

## 4. Role-specific authentication

Separate school, employee, and higher-education login concepts communicate the intended audience while keeping the authentication pattern consistent.

## 5. Government-focused visual identity

The designs balance institutional credibility with a modern digital-service aesthetic through restrained colors, structured spacing, simple iconography, and accessible contrast.
